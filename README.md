# PromoHub

Android-приложение для автоматического сбора, хранения и отображения актуальных промокодов сервисов доставки еды.

## Описание проекта

PromoHub — это клиент-серверное приложение, которое собирает промокоды из различных источников (сайты, Telegram-каналы) и предоставляет удобный интерфейс для их просмотра, поиска и использования. Пользователи помогают поддерживать актуальность: если **два разных пользователя** отметили промокод как нерабочий, он автоматически скрывается у всех.

### Поддерживаемые сервисы

- **Самокат** — доставка продуктов
- **Яндекс Еда** — доставка еды из ресторанов
- **Delivery Club** — доставка еды и продуктов
- **ВкусВилл** — продукты и готовая еда

## Технологии

### Backend

- **FastAPI** — фреймворк для REST API
- **PostgreSQL** — реляционная база данных
- **SQLAlchemy** — ORM для Python
- **Alembic** — управление миграциями
- **JWT** — аутентификация

### Android

- **Kotlin** — язык программирования
- **Jetpack Compose** — декларативный UI
- **MVVM** — архитектурный паттерн
- **Retrofit** — HTTP клиент
- **Room** — локальная база данных
- **Navigation Compose** — навигация

## Запуск проекта

### Backend

1. Установить Docker и Docker Compose
2. Запустить проект:
   ```bash
   docker-compose up --build
   ```
3. Сервер будет доступен по адресу: `http://localhost:8000`
4. Swagger документация: `http://localhost:8000/docs`

### Android

1. Открыть проект в Android Studio
2. Подождать синхронизации Gradle
3. Запустить на эмуляторе или устройстве

> Примечание: Для работы Android-приложения необходимо запущенное backend-приложение. По умолчанию используется адрес `http://10.0.2.2:8000/` (эмулятор Android). На реальном телефоне адрес сервера задаётся прямо в приложении: **Профиль → Адрес сервера**. Это позволяет указать публичный адрес туннеля до домашнего ПК без пересборки APK.
>
> Как поднять сервер на своём ПК и открыть к нему доступ из любой сети (например, из университета) — подробно описано в [docs/server_setup.md](docs/server_setup.md).

## API Endpoints

### Аутентификация

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/auth/register` | Регистрация пользователя |
| POST | `/api/auth/login` | Вход в систему |
| GET | `/api/auth/me` | Получение текущего пользователя |

### Промокоды

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/promocodes/` | Список промокодов |
| GET | `/api/promocodes/{id}` | Детали промокода |
| POST | `/api/promocodes/` | Создание промокода |
| PUT | `/api/promocodes/{id}` | Обновление промокода |
| DELETE | `/api/promocodes/{id}` | Удаление промокода |
| POST | `/api/promocodes/{id}/vote` | Голос «работает / не работает» (2 голоса «не работает» от разных пользователей скрывают код) |
| POST | `/api/promocodes/{id}/favorite` | Добавление/удаление из избранного |
| GET | `/api/promocodes/favorites/list` | Список избранных |
| GET | `/api/promocodes/history/list` | История просмотров |
| DELETE | `/api/promocodes/history/clear` | Очистка истории |

## Функционал приложения

- Просмотр списка промокодов
- Поиск по названию и описанию
- Фильтрация по сервисам
- Просмотр детальной информации
- Копирование промокода в буфер обмена
- Добавление в избранное
- История просмотров
- Голосование «работает / не работает» (нерабочие коды скрываются у всех)
- Авторизация и регистрация
- Профиль пользователя с настройкой адреса сервера
