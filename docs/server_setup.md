# Запуск сервера и доступ из университета (туннель)

Сценарий: **сервер (твой ПК) стоит дома, телефон с приложением — в университете, общей сети нет.**
Чтобы приложение работало «из интернета с твоего ПК», поднимаем на ПК backend и пробрасываем его наружу
через туннель, который даёт публичный `https`-адрес. Этот адрес ты вставляешь в приложении один раз.

```
[Телефон в универе] --internet--> [https://xxxx.trycloudflare.com] --туннель--> [твой ПК: localhost:8000]
```

---

## Шаг 1. Поднять backend на ПК (дома)

Есть два способа. Если Docker **не установлен** — используй Вариант A (проще и быстрее).

### Вариант A — без Docker (Python + SQLite) ✅ рекомендуется, если нет Docker

Нужен установленный Python (он у тебя есть). Одной командой:

```powershell
powershell -ExecutionPolicy Bypass -File C:\Users\яр\Documents\parser\backend\run-local.ps1
```

Скрипт сам создаст окружение, поставит зависимости и запустит сервер на `http://localhost:8000`.
Окно **не закрывай** — пока оно открыто, сервер работает. База — файл `backend\promohub.db`.

> Чтобы пересоздать промокоды (например, после смены сервисов) — закрой сервер (Ctrl+C),
> удали файл `backend\promohub.db` и запусти скрипт снова.

### Вариант B — через Docker

Нужен установленный и запущенный **Docker Desktop**.

```powershell
cd C:\Users\яр\Documents\parser
docker compose down -v
docker compose up --build -d
```

> `down -v` очищает том БД — это нужно при первом запуске после смены сервисов, иначе останутся
> старые промокоды (сид-данные заполняют только пустую базу).

### Проверка (для любого варианта)

- Открой в браузере http://localhost:8000/health → должно быть `{"status":"ok"}`
- Swagger с описанием API: http://localhost:8000/docs

---

## Шаг 2. Поднять туннель (даёт публичный https-адрес)

### Вариант A — Cloudflare Tunnel (рекомендуется, бесплатно, без регистрации)

1. Скачай `cloudflared` для Windows: https://github.com/cloudflare/cloudflared/releases
   (файл `cloudflared-windows-amd64.exe`, переименуй в `cloudflared.exe`).
2. Запусти быстрый туннель:
   ```powershell
   .\cloudflared.exe tunnel --url http://localhost:8000
   ```
3. В консоли появится строка вида:
   ```
   https://random-words-1234.trycloudflare.com
   ```
   Это и есть публичный адрес твоего сервера. Скопируй его.

Либо просто запусти готовый скрипт из репозитория (см. `scripts/start-tunnel.ps1`).

### Вариант B — ngrok

1. Установи ngrok: https://ngrok.com/download , зарегистрируйся, добавь authtoken.
2. Запусти:
   ```powershell
   ngrok http 8000
   ```
3. Скопируй адрес `Forwarding` вида `https://xxxx.ngrok-free.app`.

---

## Шаг 3. Указать адрес в приложении

1. Открой приложение на телефоне → вкладка **Профиль**.
2. В блоке **«Адрес сервера»** вставь публичный адрес из шага 2
   (например `https://random-words-1234.trycloudflare.com`).
3. Нажми **«Сохранить адрес»**. Адрес сохраняется на телефоне — повторно вводить не нужно,
   пока он не сменится.
4. Вернись на **Главную** и потяни/обнови — промокоды подгрузятся с твоего ПК.

> Адрес `trycloudflare.com` меняется при каждом перезапуске туннеля. Если перезапустил туннель —
> просто вставь новый адрес в Профиле (пересобирать APK не нужно).

## Чек-лист перед защитой

- [ ] Дома: ПК включён, не уходит в сон (Параметры → Питание → «Никогда» для сна).
- [ ] Дома: `docker compose up -d` запущен, http://localhost:8000/health отвечает.
- [ ] Дома: туннель запущен, адрес скопирован.
- [ ] В приложении (Профиль) вставлен актуальный адрес туннеля, нажато «Сохранить».
- [ ] Проверил с мобильного интернета (выключи Wi-Fi дома и открой адрес туннеля в браузере телефона
      `https://.../docs` — Swagger должен открыться). Если открывается — приложение тоже достучится.

## Возможные проблемы

| Симптом | Причина | Решение |
|---|---|---|
| В приложении «Ошибка загрузки данных» | Неверный/старый адрес туннеля | Вставь актуальный адрес в Профиле |
| Туннель открывается, но `/health` не отвечает | Backend не запущен | `docker compose up -d` |
| Список промокодов пустой | База осталась от старой версии | `docker compose down -v` и заново |
| ПК ушёл в сон в универе — всё отвалилось | Сон/гибернация дома | Отключить сон в настройках питания |
