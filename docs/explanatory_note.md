# Пояснительная записка

## Курсовой проект: PromoHub

### 1. Введение

Данный проект представляет собой Android-приложение «PromoHub» — систему автоматического сбора, хранения и отображения актуальных промокодов для популярных сервисов доставки еды: Самокат, Яндекс Еда, Delivery Club и ВкусВилл. Особенность системы — краудсорсинговая проверка актуальности: если два разных пользователя отмечают промокод как нерабочий, он автоматически скрывается у всех.

#### 1.1. Актуальность

В современном мире онлайн-шопинга промокоды являются важным инструментом для привлечения клиентов и получения скидок. Однако количество источников промокодов велико, и отслеживать их вручную неудобно. Приложение PromoHub решает эту проблему, автоматически собирая промокоды и предоставляя удобный интерфейс для их использования.

#### 1.2. Цель и задачи

**Цель:** Разработка мобильного приложения для агрегации промокодов.

**Задачи:**
1. Разработка серверной части на FastAPI
2. Создание Android-приложения с использованием Jetpack Compose
3. Реализация авторизации и управления промокодами
4. Интеграция с PostgreSQL для хранения данных
5. Создание документации

### 2. Технологический стек

#### 2.1. Backend (серверная часть)

- **FastAPI** — асинхронный фреймворк для создания REST API на Python. Выбран за высокую производительность и автоматическую генерацию Swagger-документации.
- **PostgreSQL** — объектно-реляционная система управления базами данных. Выбрана за надёжность, поддержку JSON и расширенные возможности запросов.
- **SQLAlchemy** — ORM (Object-Relational Mapping) для Python. Предоставляет удобный интерфейс для работы с базой данных.
- **Alembic** — инструмент для управления миграциями базы данных SQLAlchemy.
- **JWT (JSON Web Tokens)** — стандарт для безопасной передачи информации между клиентом и сервером.

#### 2.2. Android (клиентская часть)

- **Kotlin** — современный язык программирования для Android, совместимый с Java.
- **Jetpack Compose** — декларативная система для создания пользовательского интерфейса. Выбрана за современный подход и упрощение разработки UI.
- **MVVM (Model-View-ViewModel)** — архитектурный паттерн для отделения логики от представления.
- **Retrofit** — типобезопасный HTTP-клиент для Android и Java.
- **Room** — библиотека для работы с базой данных SQLite на Android.

### 3. Архитектура приложения

#### 3.1. Общая архитектура

Приложение построено по архитектуре «клиент-сервер»:

- **Клиент** — Android-приложение, отвечающее за отображение данных и взаимодействие с пользователем.
- **Сервер** — FastAPI-приложение, обрабатывающее бизнес-логику и работающее с базой данных.

#### 3.2. Архитектура Android-приложения

Android-приложение следует паттерну MVVM с разделением на слои:

1. **Data Layer (Слой данных):**
   - `RetrofitClient` — клиент для HTTP-запросов к серверу
   - `ApiService` — интерфейс API с описанием эндпоинтов
   - `AppDatabase` — локальная база данных Room
   - `PromoCodeDao` — DAO для работы с таблицей промокодов
   - `TokenManager` — менеджер для хранения токена авторизации
   - `SettingsManager` — хранение пользовательских настроек (адрес сервера)
   - `PromoRepository` — репозиторий, объединяющий данные из сети и локальной БД

2. **Domain Layer (Слой бизнес-логики):**
   - `AuthViewModel` — ViewModel для экрана авторизации
   - `PromoViewModel` — ViewModel для работы с промокодами

3. **UI Layer (Слой представления):**
   - Экраны на Jetpack Compose: Login, Home, Detail, Favorites, History, Profile
   - Компоненты: `PromoCodeCard`, `ServiceDropdown` (выплывающий список сервисов)

#### 3.3. Архитектура Backend

Backend построен на фреймворке FastAPI с разделением на модули:

1. **Routers (Маршруты):**
   - `auth.py` — эндпоинты аутентификации
   - `promocodes.py` — эндпоинты для работы с промокодами

2. **Core (Ядро):**
   - `main.py` — точка входа приложения
   - `database.py` — конфигурация подключения к БД
   - `models.py` — SQLAlchemy модели
   - `schemas.py` — Pydantic схемы
   - `auth.py` — модуль аутентификации
   - `crud.py` — CRUD операции
   - `seed.py` — начальные данные

### 4. База данных

#### 4.1. Структура базы данных

База данных PostgreSQL содержит следующие таблицы:

1. **users** — пользователи
   - id (PK)
   - username (уникальное)
   - email (уникальное)
   - hashed_password
   - avatar_url
   - created_at

2. **promocodes** — промокоды
   - id (PK)
   - code
   - title
   - description
   - service
   - discount
   - expires_at
   - is_active
   - rating
   - usage_count
   - source_url
   - created_at
   - updated_at

3. **favorites** — избранное
   - id (PK)
   - user_id (FK)
   - promocode_id (FK)
   - created_at

4. **history** — история просмотров
   - id (PK)
   - user_id (FK)
   - promocode_id (FK)
   - viewed_at

5. **promo_votes** — голоса пользователей о работоспособности промокода
   - id (PK)
   - user_id (FK)
   - promocode_id (FK)
   - is_working (BOOLEAN — true: работает, false: не работает)
   - created_at
   - UNIQUE(user_id, promocode_id) — один голос на пользователя

#### 4.2. Связи между таблицами

- Один пользователь может иметь много избранных промокодов (1:N)
- Один промокод может быть в избранном у многих пользователей (N:M через favorites)
- Один пользователь может просматривать много промокодов (1:N через history)
- Один пользователь оставляет один голос на промокод (ограничение UNIQUE в promo_votes); при наборе двух голосов «не работает» от разных пользователей промокод скрывается (is_active = false)

### 5. REST API

#### 5.1. Эндпоинты аутентификации

| Метод | Путь | Описание | Авторизация |
|-------|------|----------|-------------|
| POST | /api/auth/register | Регистрация | Нет |
| POST | /api/auth/login | Вход | Нет |
| GET | /api/auth/me | Текущий пользователь | Да |

#### 5.2. Эндпоинты промокодов

| Метод | Путь | Описание | Авторизация |
|-------|------|----------|-------------|
| GET | /api/promocodes/ | Список промокодов | Да |
| GET | /api/promocodes/{id} | Детали промокода | Да |
| POST | /api/promocodes/ | Создание | Да |
| PUT | /api/promocodes/{id} | Обновление | Да |
| DELETE | /api/promocodes/{id} | Удаление | Да |
| POST | /api/promocodes/{id}/vote | Голос «работает/не работает» | Да |
| POST | /api/promocodes/{id}/favorite | Избранное | Да |
| GET | /api/promocodes/favorites/list | Список избранного | Да |
| GET | /api/promocodes/history/list | История | Да |
| DELETE | /api/promocodes/history/clear | Очистка истории | Да |

### 6. Реализованный функционал

#### 6.1. Авторизация
- Регистрация нового пользователя
- Вход в систему с получением JWT-токена
- Сохранение токена в DataStore
- Выход из аккаунта

#### 6.2. Просмотр промокодов
- Список всех активных промокодов
- Детальная информация о промокоде
- Копирование промокода в буфер обмена
- Отображение количества голосов «работает» и «не работает»

#### 6.3. Поиск и фильтрация
- Поиск по названию и описанию
- Фильтрация по сервисам через выплывающий список (Самокат, Яндекс Еда, Delivery Club, ВкусВилл)

#### 6.3.1. Голосование за работоспособность
- Кнопки «Работает» / «Не работает» на экране деталей
- Учёт уникальных голосов пользователей
- Автоматическое скрытие промокода у всех при двух голосах «не работает» от разных пользователей

#### 6.4. Избранное
- Добавление промокода в избранное
- Удаление из избранного
- Просмотр списка избранных промокодов

#### 6.5. История
- Автоматическое сохранение просмотренных промокодов
- Просмотр истории
- Очистка истории

#### 6.6. Профиль
- Отображение информации о пользователе
- Настройка адреса сервера (для подключения к серверу через публичный туннель без пересборки APK)
- Выход из аккаунта
