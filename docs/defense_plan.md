# План защиты проекта PromoHub

## 1. Вступительное слово (2-3 минуты)

### Тема проекта
«PromoHub — Android-приложение для автоматического сбора и отображения промокодов»

### Актуальность
- Рост онлайн-шопинга и популярность промокодов
- Необходимость агрегации информации из разных источников
- Удобство мобильного доступа к промокодам

### Цель проекта
Разработка мобильного приложения для агрегации промокодов с серверной частью и базой данных

## 2. Обзор технологий (3-4 минуты)

### Backend
- **FastAPI** — почему выбран (производительность, автодокументация)
- **PostgreSQL** — надёжность, расширяемость
- **SQLAlchemy** — удобство работы с БД
- **JWT** — безопасность авторизации

### Android
- **Kotlin** — современный язык для Android
- **Jetpack Compose** — декларативный подход к UI
- **MVVM** — разделение логики и представления
- **Retrofit** — работа с сетью
- **Room** — локальное хранение данных

## 3. Архитектура решения (5-6 минут)

### Общая схема
- Клиент-серверная архитектура
- REST API для коммуникации
- JWT для аутентификации

### Android-приложение
Показать схему слоёв:
1. **Data Layer** — Retrofit, Room, Repository
2. **Domain Layer** — ViewModels
3. **UI Layer** — Compose screens

### Backend
Показать схему модулей:
1. **Routers** — обработка HTTP-запросов
2. **CRUD** — операции с базой данных
3. **Auth** — аутентификация и авторизация
4. **Models** — SQLAlchemy модели

## 5. Технические особенности (3-4 минуты)

### Ключевые решения
1. **Кэширование данных** — Room для кэширования промокодов
2. **Автоматическое скрытие** — удаление старых промокодов через 30 дней
3. **JWT-аутентификация** — безопасное хранение токена
4. **Краудсорсинговая проверка** — голосование «работает/не работает»; промокод скрывается у всех при двух голосах «не работает» от разных пользователей

### Масштабируемость
- Добавление новых сервисов через seed-данные
- Расширение API новыми эндпоинтами
- Возможность добавления push-уведомлений

## 6. Ответы на вопросы (5 минут)

### Возможные вопросы:
1. **Почему выбран FastAPI, а не Django?**
   - Высокая производительность асинхронных запросов
   - Автоматическая генерация Swagger-документации
   - Современный подход к разработке API

2. **Как обеспечивается безопасность?**
   - JWT-токены для аутентификации
   - Хеширование паролей через bcrypt
   - Валидация входных данных через Pydantic

3. **Как работает кэширование?**
   - Данные сохраняются в Room (SQLite)
   - При отсутствии сети отображаются кэшированные данные
   - При наличии сети данные обновляются

4. **Как добавить новый сервис?**
   - Добавить запись в seed.py
   - Указать цвет в Android-приложении
   - Обновить список сервисов в UI

5. **Каково максимальное количество промокодов?**
   - Система поддерживает неограниченное количество
   - Реализована пагинация (limit/offset)
